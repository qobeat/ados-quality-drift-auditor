1. Can the auditor distinguish Project from Package?
2. Can it distinguish Archetype from Domain?
3. Can it reconstruct the relationship from User Request to Intent, Goal, and Objectives?
4. Can it explain which ADOS Profile concepts and Pillars apply?
5. Can it distinguish Prompt, Directive, Work, Atomic Work, Claim, and Requirement?
6. Can it distinguish Apply Directive from Verify Directive and QA Evidence?
7. Can it evaluate a PDD without requiring every Project to have one?
8. Can it identify whether Build Package, Building Agent Work, and Build Result are all available?
9. Can it identify where drift first appeared?
10. Can it compare a newer version against an older but stronger Baseline Version?
11. Can it represent mixed movement across Goal-Attractor Coordinates?
12. Can it distinguish Metric from Coordinate Dimension and Pillar?
13. Can it prevent a high score from replacing semantic judgment?
14. Can it detect schema coverage without schema strength?
15. Can it detect a proof fixture generalized beyond its direct tests?
16. Can it evaluate an Evidence Ledger or Execution Trace when present without requiring one?
17. Can it identify app-specific requirements incorrectly applied to Research, Runtime, Profile, or Prompt Projects?
18. Can it cite glossary terms used in every major finding?
19. Can it distinguish observed consequence from future Drift Risk?
20. Can it produce a focused Corrective Prompt that preserves Goal and Objectives?
