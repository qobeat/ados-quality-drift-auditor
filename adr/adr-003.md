## Status

Accepted.

## Context

Build prompts, execution traces, software packages, knowledge models, and control packs do not have identical quality obligations. A single generic score misses archetype-specific risks, while unrestricted profile combinations permit metric shopping.

## Decision

1. Select exactly one primary archetype from the predefined bank.
2. Use archetype-adjusted metrics for the three input stages.
3. Use the fixed nine common axes for cross-source comparison.
4. Record modifiers without creating a second primary weighting frame.
5. Use the general profile when no specialized profile reaches 0.60 confidence.
6. Keep weights at 100 per stage and common frame.

## Consequences

- Schema strength matters more for knowledge packages.
- Function, replay, and resilience matter more for runtime artifacts.
- Controls and agent work remain separately scoreable.
- Cross-version comparison remains possible through the common axes.
