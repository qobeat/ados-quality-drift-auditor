## Context

A quality-drift auditor needs a stable semantic spine before schemas and metrics can be reliable.

## Decision

The v0.2 glossary includes:

1. User Request, Intent, Goal, Objective, Project, Package, Archetype, Domain, ADOS Profile, Flow, Skill, and Glossary.
2. Build Package, Building Agent Work, Build Result, Baseline Version, Quality Drift, consequence, risk, Problem Pattern, and Corrective Prompt.
3. Prompt, Directive, Delivery, Work, Apply Directive, Verify Directive, PDD, Statement, Claim, Atomic Claim, Requirement, Project Requirement, Atomic Requirement, and Atomic Work.
4. Goal-Attractor Geometry, Coordinate concepts, Metrics, Pillars, and the full twenty-pillar set.
5. Core-file, ADR, Evidence Ledger, Execution Trace, and QA Evidence concepts.
6. Six Archetypes and seven Domains.

## Alternatives

- Keep a small generic evaluation dictionary.
- Import every source glossary term unchanged.

## Consequences

- The glossary is large but directly aligned with Project logic.
- Source terms are reviewed for applicability.
- Generic bookkeeping terms remain excluded unless given an ADOS-specific boundary.
