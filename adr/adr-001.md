## Status

Accepted.

## Context

ADOS Glossary v1.3 was the strongest evaluated semantic and schema baseline among the supplied evolution artifacts. Later versions added useful structure but also demonstrated that structural growth can weaken contracts.

## Decision

This package adopts these patterns:

1. closed JSON Schema contracts for governed records;
2. stable identifiers for claims, evidence, sources, versions, metrics, problems, and comparisons;
3. exact file inventory and SHA-256 integrity records;
4. explicit authority load order;
5. human-readable and machine-readable companion surfaces;
6. validation of the final extracted package.

The v1.3 project is a design source, not hidden runtime authority.

## Consequences

- Adding fields requires an explicit schema change.
- Reports cannot silently add ad hoc properties.
- Package validation can detect unregistered files and weakened machine surfaces.
- Minimal v0.1 files may be extended in v1.0 only through governed revisions.
