## Context

The auditor must interpret ADOS-specific concepts consistently across Build Packages, agent work, results, versions, Metrics, and Problem Patterns.

## Decision

`GLOSSARY.md` is the primary semantic authority. Banks, schemas, skills, docs, and reports reference stable glossary term IDs and must not redefine those terms locally.

## Alternatives

- Allow each bank to define its own terminology.
- Use schemas as the semantic authority.
- Use source packages directly as runtime ontology.

## Consequences

- The glossary is intentionally comprehensive.
- Retrieval of relevant terms precedes scoring.
- Schema validity cannot substitute for semantic consistency.
- Source definitions are adapted before becoming glossary authority.

## Scope

`GLOSSARY.md`, skill, profiles, metrics, problems, schemas, reports.
